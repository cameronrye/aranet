## Why Aranet Rust?

<CardGrid>
  <Card title="Complete BLE Stack" icon="rocket">
    Full Bluetooth Low Energy support via btleplug - scan, connect, read current values, download history, configure settings.
  </Card>
  <Card title="Multi-Platform" icon="laptop">
    Works on macOS, Windows, and Linux. Native performance with Rust's zero-cost abstractions.
  </Card>
  <Card title="Real-Time Streaming" icon="random">
    Subscribe to live sensor updates with auto-reconnection and configurable backoff strategies.
  </Card>
  <Card title="Comprehensive Tooling" icon="setting">
    CLI for quick readings, TUI dashboard for monitoring, GUI app for desktop, and background service with REST API.
  </Card>
</CardGrid>

## Quick Start

<Tabs>
  <TabItem label="Homebrew (Recommended)">
```bash
# macOS and Linux - Recommended
brew tap cameronrye/aranet
brew install aranet

# Verify installation
aranet --version
```
  </TabItem>
  <TabItem label="Shell Script">
```bash
# macOS and Linux
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/cameronrye/aranet/releases/latest/download/aranet-cli-installer.sh | sh

# Verify installation
aranet --version
```
  </TabItem>
  <TabItem label="PowerShell">
```powershell
# Windows
irm https://github.com/cameronrye/aranet/releases/latest/download/aranet-cli-installer.ps1 | iex

# Verify installation
aranet --version
```
  </TabItem>
  <TabItem label="Cargo">
```bash
# From crates.io (requires Rust toolchain)
cargo install aranet-cli

# Verify installation
aranet --version
```
  </TabItem>
  <TabItem label="Source">
```bash
# Build from source
git clone https://github.com/cameronrye/aranet.git
cd aranet && cargo build --release

# Binary will be at ./target/release/aranet
./target/release/aranet --version
```
  </TabItem>
</Tabs>

<Aside type="tip" title="Quick Tip">
After installation, run `aranet doctor` to check your Bluetooth setup and permissions.
</Aside>

<Tabs>
  <TabItem label="Scan">
```bash
# Find nearby Aranet devices
aranet scan

# Output:
# Found 2 devices:
#   Aranet4 1234A  AA:BB:CC:DD:EE:FF  -45 dBm
#   Aranet2 5678B  11:22:33:44:55:66  -52 dBm
```
  </TabItem>
  <TabItem label="Read">
```bash
# Read current sensor values
aranet read AA:BB:CC:DD:EE:FF

# Output:
# CO₂: 847 ppm (Green)
# Temperature: 22.4°C
# Humidity: 45%
# Pressure: 1013 hPa
```
  </TabItem>
  <TabItem label="History">
```bash
# Download measurement history
aranet history AA:BB:CC:DD:EE:FF --output history.csv

# Filter by date range
aranet history AA:BB:CC:DD:EE:FF --since 2026-01-15

# Export as JSON
aranet history AA:BB:CC:DD:EE:FF --format json
```
  </TabItem>
  <TabItem label="Multi-Device">
```bash
# Setup device aliases for easy access
aranet alias set living-room AA:BB:CC:DD:EE:FF
aranet alias set bedroom 11:22:33:44:55:66

# Read from multiple devices
aranet read -d living-room -d bedroom

# Watch all aliased devices
aranet watch --all
```
  </TabItem>
</Tabs>

### Core Libraries

<CardGrid>
  <Card title="aranet-types" icon="document">
    Platform-agnostic data types shared across all crates - readings, device info, errors.
  </Card>
  <Card title="aranet-core" icon="puzzle">
    BLE communication via btleplug - discovery, readings, history, settings, streaming.
  </Card>
  <Card title="aranet-store" icon="seti:db">
    SQLite persistence for offline access - incremental sync, deduplication, queries.
  </Card>
</CardGrid>

### Applications

<CardGrid>
  <Card title="aranet-cli" icon="seti:shell">
    Command-line interface with multi-device support, aliases, passive mode, CSV export.
  </Card>
  <Card title="aranet-tui" icon="laptop">
    Terminal dashboard with sparklines, alerts, vim keys, multi-device monitoring.
  </Card>
  <Card title="aranet-service" icon="setting">
    Background daemon with REST API, Prometheus metrics, MQTT publishing, WebSocket streaming.
  </Card>
  <Card title="aranet-gui" icon="approve-check">
    Native desktop application with real-time monitoring, charts, and multi-device support.
  </Card>
</CardGrid>
